### 🚀 Today I Learned

#### | 추상 클래스

`추상 클래스`란, 클래스 간의 계층구조에서 상위에 존재하며 하위 클래스를 대표하는 클래스이다.

- 추상 클래스는 직접 혼자 객체화가 될 수 없다. (생성자를 사용하여 객체 생성 불가)
- 추상 클래스는 다른 클래스에 의해 상속되어야 한다.

다시 말해서, 추상의 의미대로 "구체적"이지 않은 클래스로 보다 구체적인 하위 클래스를 대표하는 클래스이다.

```
public abstract class 클래스명 {
  private 타입 변수명;

  abstract void 메소드 명( );
}
```

하위 클래스는 extends 를 통해 Abstract 클래스를 상속받는다.
만약 Abstract 클래스 안에 abstract method가 있다면 꼭 하위 클래스에서 해당 method를 구현해야 한다.

- 클래스 Shape은 계층구조 상위에 있어 전체 클래스를 대표
- 클래스 자체만이 객체로 생성될 필요 없는 클래스
- 클래서 Shape는 아직 도형의 성질이 구제척이지 낳으며 다른 도형의 공통적인 성질만을 갖고 있다.
  ⇒ Shape를 추상 클래스로 정의

#### | 추상 클래스 사용 이유

- 공통된 필드와 메서드를 통일활 목적
  필드와 메서드 이름을 통일하여 유지보수성을 높이고 통일성 유지 가능
- 실체 클래스 구현 시, 시간 절역  
   강제로 주어지는 필드와 메서드를 가지고 구현
- 규격에 맞는 실체 클래스 구현  
   설계 규격에 맞게 클래스를 구현 가능하여 유지보수성을 높일 수 있음

⇒ 추상 클래스는 그 추상 클래스를 상속받아 기능을 이용하고 확장
